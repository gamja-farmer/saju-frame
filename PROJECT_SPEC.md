# PROJECT_SPEC.md  
대만 대상 사주팔자 사이드 프로젝트 스펙 정리

---

## 1. 프로젝트 개요

- 프로젝트 주제: 사주팔자(四柱八字) 기반 운세/해석 서비스
- 목표: 월 약 10만 원 수준의 소규모이지만 실제 수익 발생
- 성격: 개인 사이드 프로젝트, 저비용·저위험 실험
- 핵심 철학:  
  - “사주 결과”보다 “사주를 이해하게 만드는 해석 프레임”  
  - 정확함보다 납득감·공감·체류 시간 중시

---

## 2. 타깃 시장

- 국가: 대만
- 언어: 번체 중국어 (Traditional Chinese)
- 문화적 전제:
  - 명리/사주에 대한 거부감이 낮음
  - 텍스트 기반 해석에 익숙
  - 감정·관계·인생 흐름 중심 콘텐츠 선호
- 주 사용자:
  - 20~40대
  - 연애, 커리어, 인간관계 고민이 많은 층
  - “정확한 계산”보다 “나를 이해해주는 설명”에 반응

---

## 3. 벤치마크 & 차별화

### 3.1 벤치마크

- 한국의 청월아씨 사주 페이지
- 참고 요소:
  - 질문 → 공감 → 해석으로 이어지는 흐름
  - 결과를 단정하지 않고 해석의 여지를 남기는 문장
  - 감성적이지만 과도하지 않은 UX

### 3.2 대만 기존 사주 사이트의 한계

- 입력 즉시 결과 출력 (몰입도 낮음)
- 한 화면에 정보 과다
- 표·한자·전문 용어 중심
- “맞다/틀리다” 판단을 유저에게 강요

### 3.3 본 서비스의 차별점

- 즉답형 결과 배제
- 해석을 단계적으로 분해
- “당신이 이런 사람일 가능성이 높다”는 구조
- 스스로 고개를 끄덕이게 만드는 UX

---

## 4. 핵심 UX 전략

1. 계산 결과를 앞에 두지 않는다  
2. 사용자의 상태를 먼저 언어화한다  
3. 사주 용어는 뒤로 숨기고 해석을 전면에 배치한다  
4. 모든 문장은 “아마도 / 경향적으로 / 이런 흐름을 보인다” 형태 유지  
5. 체류 시간이 곧 신뢰도라는 전제

---

## 5. 페이지 구조 (텍스트 와이어프레임 기준)

### 5.1 진입 페이지 (Landing)

- 헤드라인  
  - “你是不是也有這種感覺？”
- 공감 문장 2~3줄
- 이 서비스가 해주는 것
  - 성격을 단정하지 않음
  - 인생의 흐름을 설명함
- CTA
  - “查看你的命盤解讀”

---

### 5.2 정보 입력 페이지

- 최소 입력:
  - 생년월일
  - 출생 시간 (선택)
  - 성별 (선택)
- 입력 중간중간 설명 문구 삽입
  - “不知道也沒關係”
- 버튼:
  - “開始解讀”

---

### 5.3 해석 결과 페이지 (핵심)

1단계: 전체 인상  
- “你的命盤給人的第一個感覺是…”

2단계: 성향 프레임  
- 에너지 사용 방식
- 관계에서의 패턴
- 선택을 미루는/빠른 이유

3단계: 삶의 흐름  
- 강해지는 시기
- 흔들리는 시기
- 반복되는 테마

4단계: 사주 용어 보조 설명 (접기/펼치기)
- 오행
- 십성
- 간단한 관계 설명

---

## 6. 사주 로직 설계 원칙

- AI 사용 최소화
- 외부 API 의존 배제
- 규칙 기반 / 계산 기반 구조
- 사주 타입은 약 10~15개 내외로 제한
- 결과 다양성은 “문장 조합”으로 확보

---

## 7. 콘텐츠 전략 (SEO)

- 공감형 블로그 글 → 결과 페이지 유입
- 예시 주제:
  - “為什麼你總是做決定很慢？”
  - “命盤裡容易累的人有什麼特徵？”
- 글 구조:
  - 상황 공감
  - 일반적 설명
  - 사주 관점 해석
  - 서비스 CTA 연결

---

## 8. 수익화 전략

### 8.1 초기

- 페이지 하단 광고 배너
- 과도한 노출 금지
- 신뢰도 우선

### 8.2 추후 가능성

- 상세 해석 잠금 해제 (소액)
- 특정 테마 유료 해석
- 광고 제거 옵션

---

## 9. 성공 판단 기준

- 방문 대비 결과 페이지 도달률
- 평균 체류 시간
- 재방문율
- “정확하다”보다 “이해된다” 반응

---

## 10. 핵심 요약 문장

- 이 서비스는 사주를 맞히는 서비스가 아니다  
- 사용자가 자기 인생을 설명할 언어를 빌려주는 서비스다
